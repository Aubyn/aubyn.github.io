d.ts 自动引用 ​

md
typescript 中，为何有些 d.ts 文件能自动被全局自动引用

在 TypeScript 中，有些 .d.ts（类型声明）文件能够被全局自动引用，主要是由 “TypeScript 的模块机制” 和 “编译器的文件发现规则” 共同决定的。 具体来说，有以下几个核心原因：

1. 文件内没有顶层的 import 或 export（最核心原因） ​

在 TypeScript 中，判断一个文件是“模块”还是“全局脚本”的唯一标准是：该文件中是否存在顶层的 import 或 export 关键字。

  • 全局脚本文件： 如果一个 .d.ts 文件中没有任何顶层 import 或 export（即使在 declare global 之外），TypeScript 就会把它当作全局脚本文件。它里面声明的所有类型、变量都会自动挂载到全局作用域中。
    typescript
    // global.d.ts
declare const myGlobalVar: string; // 全局可用，无需 import
interface Window {
  myCustomProp: number;
} // 全局可用，无需 import
  • 模块文件： 如果一个 .d.ts 文件哪怕只有一行 export {}，它就变成了一个模块。它里面的声明就不会污染全局，必须通过 import 才能使用。

2. tsconfig.json 中的 include 配置 ​

TypeScript 编译器需要知道去哪里找这些文件。如果你的 tsconfig.json 配置了 include，并且匹配到了这些全局 .d.ts 文件，它们就会被编译器自动加载。

json
{
  "compilerOptions": {
    // ...
  },
  "include": ["src/**/*", "types/**/*.d.ts"]
}

只要 types 目录下的 .d.ts 符合“全局脚本文件”的特征，它们里面的类型就会被全局自动注入。

3. 默认的 @types 自动加载机制 ​

TypeScript 默认会自动将 node_modules/@types 目录下的所有包引入到项目中。 例如，当你安装了 @types/node 后，你不需要在代码里写 import { process } from 'node' 就能直接使用 process。 这是因为 @types/node 中的某些 .d.ts 文件被定义为了全局脚本文件，并且 TypeScript 默认会扫描 node_modules/@types 目录，将这些全局声明自动挂载。

你可以通过 tsconfig.json 中的 types 选项来控制这一行为。如果不配置 types，TS 会自动包含 @types 下的所有内容；如果配置了 types: ["node"]，则只会自动引入 node 的类型。

4. NPM 包 package.json 中的 types/typings 字段 ​

当你在项目中 npm install 一个第三方库时，如果这个库的 package.json 中配置了 types 或 typings 字段指向了一个 .d.ts 文件，TypeScript 就会自动识别这个类型声明文件。 如果这个包的类型文件本身是通过“全局脚本”的方式编写的（虽然现代库很少这么做，但在早期 jQuery 等库很常见），那么只要项目依赖了它，它的类型就会自动全局可用。

5. 三斜线指令 ​

这是一种较老但仍被支持的机制。在一个 .d.ts 文件或普通 .ts 文件顶部，可以通过三斜线指令显式地告诉编译器去引入其他依赖的类型：

typescript
/// <reference types="node" />
/// <reference path="../types/my-custom.d.ts" />

一旦编译器遇到这个指令，就会把对应的类型声明文件拉进来。如果被引入的文件又是全局脚本格式，那么它的类型也就全局可用了。 ​

总结 ​

简单来说，一个 .d.ts 能被全局自动引用，必须同时满足两个条件：

  1. 编译器能找到它（通过 include、@types 默认机制、或 reference 指令）。
  2. 文件内没有顶层 export/import，使其被判定为全局脚本。 最佳实践提示： 虽然全局自动引用很方便，但现代 TypeScript 开发强烈推荐使用模块化（ES Modules），即在 .d.ts 文件中通过 export 导出类型，在使用的地方通过 import 引入。这样可以避免全局命名冲突，让类型依赖关系更加清晰，也有利于 Tree-shaking 和长期维护。仅对一些不可避免的全局变量（如扩展 Window、注入到 HTML 中的第三方 SDK）才使用全局 .d.ts。

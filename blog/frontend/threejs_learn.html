Three.js 30天学习计划 — 企业3D数据可视化方向 ​

适用人群： 5年前端工程师，3D零基础 学习场景： 工作空余时间，每天约1.5~2h（部分难点拆多天） 总预算： 约50~60小时 核心原则： 80/20 — 只学企业数据可视化高频用到的20%知识，覆盖80%工作场景

📊 难度与时间说明 ​

标记难度预估时长说明
🟢入门1~1.5h概念理解 + 跟着写一遍就能掌握
🟡中等1.5~2h需要理解原理 + 动手练习
🔴较难2~3h概念多/坑多，建议拆2天消化
🔄复习/缓冲1h巩固前几天内容，或补进度

💡 关于时间： 标注的是"有效学习时间"。如果某天特别忙，可以只看文档不做练习，标记为"预习"，第二天补实操。计划中已预留3天缓冲日。

Phase 1：3D基础与渲染管线（Day 1~7） ​

目标： 建立对Three.js核心概念的理解，能搭建基础3D场景并加载外部模型。

Day 1 · 04/30 周三 — 环境搭建与第一个场景 🟢 ​

预估时长： 1~1.5h

学习内容：

  • 使用 Vite 创建项目，安装 three
  • 理解 Three.js 三件套：Scene（场景）、Camera（相机）、Renderer（渲染器）
  • 写出第一个3D场景：一个旋转的立方体
  • 理解右手坐标系、单位（1单位 = 你定义的任意长度）

学习资料：

  • Three.js 手册 - 第一步：https://threejs.org/manual/#en/fundamentals
  • Vite 官方 - 创建项目：https://vitejs.dev/guide/#scaffolding-your-first-vite-project

实操任务：

bash
npm create vite@latest threejs-demo -- --template vanilla-ts
cd threejs-demo
npm install three @types/three
npm run dev

写一个包含旋转立方体的最小场景。

Day 2 · 05/01 周四 — 几何体与材质 🟢 ​

预估时长： 1.5h

学习内容：

  • 常用内置几何体：BoxGeometry、SphereGeometry、CylinderGeometry、PlaneGeometry
  • 基础材质：MeshBasicMaterial、MeshStandardMaterial、MeshPhongMaterial
  • 材质属性：color、metalness、roughness、opacity、wireframe
  • Mesh = Geometry + Material 的组合概念

学习资料：

  • Three.js 手册 - 几何体：https://threejs.org/manual/#en/primitives
  • Three.js 手册 - 材质：https://threejs.org/manual/#en/materials

实操任务： 创建一个包含5种不同几何体和材质的场景，每个物体使用不同颜色和材质属性。

Day 3 · 05/02 周五 — 光照系统 🟡 ​

预估时长： 1.5~2h

学习内容：

  • 环境光 AmbientLight — 全局均匀照明
  • 平行光 DirectionalLight — 模拟太阳光
  • 点光源 PointLight — 模拟灯泡
  • 聚光灯 SpotLight — 手电筒效果
  • 光照辅助工具：DirectionalLightHelper、PointLightHelper
  • 阴影：renderer.shadowMap、castShadow、receiveShadow

学习资料：

  • Three.js 手册 - 光照：https://threejs.org/manual/#en/lights
  • Three.js 示例 - 各种灯光：https://threejs.org/examples/#webgl_lights_spotlights

实操任务： 搭建一个有阴影的场景：一个平面 + 一个立方体，添加 DirectionalLight 并开启阴影。

Day 4 · 05/03 周六 — 相机系统与OrbitControls 🟡 ​

预估时长： 1.5~2h

学习内容：

  • 透视相机 PerspectiveCamera：fov、aspect、near、far
  • 正交相机 OrthographicCamera：left/right/top/bottom（数据大屏常用！）
  • 两种相机的区别与适用场景
  • OrbitControls：旋转、缩放、平移
  • OrbitControls 配置：enableDamping、minDistance、maxDistance、target

学习资料：

  • Three.js 手册 - 相机：https://threejs.org/manual/#en/cameras
  • OrbitControls 文档：https://threejs.org/docs/#examples/en/controls/OrbitControls
  • OrbitControls 示例：https://threejs.org/examples/#misc_controls_orbit

实操任务：

  1. 用透视相机搭建场景，添加OrbitControls
  2. 切换为正交相机，对比两种视角差异
  3. 限制缩放范围和旋转角度

Day 5 · 05/04 周日 — 动画循环与基础交互 🟢 ​

预估时长： 1~1.5h

学习内容：

  • requestAnimationFrame 渲染循环
  • Clock 获取 deltaTime
  • 基础动画：旋转、位移、缩放
  • 窗口自适应：window.resize 事件处理
  • 简单的 Raycaster 点击检测（入门）

学习资料：

  • Three.js 手册 - 渲染循环：https://threejs.org/manual/#en/responsive
  • Raycaster 文档：https://threejs.org/docs/#api/en/core/Raycaster

实操任务：

  1. 创建3个不同颜色的球体
  2. 点击球体时改变其颜色
  3. 窗口缩放时场景自适应

Day 6 · 05/05 周一 — 模型加载（GLTF/GLB）🔴 ​

预估时长： 2~3h（难点，建议分两次完成）

学习内容：

  • GLTF/GLB 格式介绍（为什么它是Web3D的标准格式）
  • GLTFLoader 加载模型
  • DRACOLoader 压缩模型加载
  • 模型定位、缩放、旋转
  • 模型遍历：traverse() 修改子网格材质
  • 加载进度条 LoadingManager

学习资料：

  • Three.js 手册 - 加载模型：https://threejs.org/manual/#en/load-gltf
  • GLTFLoader 文档：https://threejs.org/docs/#examples/en/loaders/GLTFLoader
  • DRACOLoader 文档：https://threejs.org/docs/#examples/en/loaders/DRACOLoader

模型资源：

  • DamagedHelmet（经典测试模型）：https://github.com/KhronosGroup/glTF-Sample-Models/tree/main/2.0/DamagedHelmet
  • FlightHelmet（带纹理的完整模型）：https://github.com/KhronosGroup/glTF-Sample-Models/tree/main/2.0/FlightHelmet
  • 小建筑模型（免费）：https://poly.pizza/search?q=building
  • Three.js内置示例模型（npm包内）：three/examples/models/gltf/

实操任务：

  1. 加载一个GLTF模型到场景中
  2. 添加OrbitControls可以旋转查看
  3. 添加LoadingManager显示加载进度
  4. 用traverse()修改模型中某个部件的颜色

Day 7 · 05/06 周二 — Phase 1 复习与缓冲 🔄 ​

预估时长： 1~1.5h（或补前几天进度）

学习内容：

  • 回顾 Phase 1 所有知识点
  • 确保以下能力达标：
    • [ ] 能独立搭建 Scene + Camera + Renderer
    • [ ] 能创建几何体并应用材质
    • [ ] 能添加光照和阴影
    • [ ] 能使用 OrbitControls 交互
    • [ ] 能加载 GLTF/GLB 模型
  • 如果有未完成的内容，今天补上

没有新资料，复习即可。

Phase 2：数据可视化核心技能（Day 8~14） ​

目标： 掌握将数据映射到3D场景的核心技术，能独立完成3D图表和交互。

Day 8 · 05/07 周三 — CSS2DRenderer 标签系统 🟡 ​

预估时长： 1.5~2h

学习内容：

  • CSS2DRenderer 和 CSS2DObject
  • 在3D场景中添加HTML标签
  • 标签样式自定义（CSS）
  • 标签跟随3D物体移动
  • 标签遮挡关系处理
  • 企业应用场景：设备名称、数据标注、状态提示

学习资料：

  • CSS2DRenderer 示例：https://threejs.org/examples/#css2d_label
  • CSS2DRenderer 文档：https://threejs.org/docs/#examples/en/renderers/CSS2DRenderer

实操任务：

  1. 创建3个立方体，每个上方添加CSS2D标签显示名称
  2. 标签样式：半透明背景、圆角、小字体
  3. 旋转场景时标签跟随物体

Day 9 · 05/08 周四 — 3D柱状图（数据驱动几何体）🔴 ​

预估时长： 2~3h

学习内容：

  • 用数据数组动态创建 BoxGeometry
  • 根据数据值映射柱体高度
  • 柱体颜色映射（渐变色/分段色）
  • 坐标轴和网格线
  • 柱体底部标签（CSS2DRenderer）
  • 动画：柱体从0增长到目标高度

学习资料：

  • Three.js 手册 - 几何体参数：https://threejs.org/manual/#en/primitives
  • BufferGeometry 文档：https://threejs.org/docs/#api/en/core/BufferGeometry

数据资源：

  • 模拟数据（自建JSON）：
json
[
    { "name": "北京", "value": 320 },
    { "name": "上海", "value": 280 },
    { "name": "广州", "value": 210 },
    { "name": "深圳", "value": 260 },
    { "name": "杭州", "value": 180 }
]

实操任务： 创建一个3D柱状图，5根柱子代表5个城市的数据，带底部标签和增长动画。

Day 10 · 05/09 周五 — 3D散点图 🟡 ​

预估时长： 1.5~2h

学习内容：

  • 用 SphereGeometry 创建数据点
  • 三维坐标映射（x, y, z → 数据维度）
  • 数据点大小/颜色映射数值
  • 坐标轴刻度标注
  • 数据点悬浮提示（Raycaster + CSS2D）

学习资料：

  • Raycaster 文档：https://threejs.org/docs/#api/en/core/Raycaster
  • Raycaster 示例：https://threejs.org/examples/#webgl_interactive_raycasting_points

数据资源：

  • 模拟数据（自建JSON）：
json
[
    { "x": 10, "y": 80, "z": 30, "value": 95, "label": "样本A" },
    { "x": 60, "y": 40, "z": 70, "value": 72, "label": "样本B" },
    { "x": 30, "y": 90, "z": 50, "value": 88, "label": "样本C" }

实操任务： 创建一个3D散点图，鼠标悬浮显示数据详情。

Day 11 · 05/10 周六 — InstancedMesh 大数据量渲染 🔴 ​

预估时长： 2~3h

学习内容：

  • 为什么不能用10000个Mesh？（DrawCall问题）
  • InstancedMesh 原理：一次DrawCall渲染大量相同几何体
  • setMatrixAt() 设置每个实例的变换矩阵
  • setColorAt() 设置每个实例的颜色
  • InstancedBufferAttribute 自定义属性
  • 企业应用场景：大规模设备点位、城市建筑群、粒子效果

学习资料：

  • InstancedMesh 文档：https://threejs.org/docs/#api/en/objects/InstancedMesh
  • InstancedMesh 示例：https://threejs.org/examples/#webgl_instancing_performance
  • InstancedMesh 动态示例：https://threejs.org/examples/#webgl_instancing_dynamic

实操任务：

  1. 用 InstancedMesh 渲染1000个立方体，随机位置和颜色
  2. 对比普通Mesh和InstancedMesh的帧率差异
  3. 添加动画：让所有实例缓慢旋转

Day 12 · 05/11 周日 — Raycaster 深度交互 🟡 ​

预估时长： 1.5~2h

学习内容：

  • Raycaster 原理：从鼠标发射射线
  • intersectObjects 检测物体
  • 鼠标悬浮高亮效果（emissive）
  • 点击选中 + 信息面板
  • 悬浮提示框跟随鼠标
  • 与 InstancedMesh 的交互（instanceId）

学习资料：

  • Raycaster 文档：https://threejs.org/docs/#api/en/core/Raycaster
  • 交互示例：https://threejs.org/examples/#webgl_interactive_cubes
  • InstancedMesh交互：https://threejs.org/examples/#webgl_interactive_instances

实操任务：

  1. 鼠标悬浮物体时高亮（改变emissive颜色）
  2. 点击物体弹出信息面板
  3. 鼠标移开恢复原色

Day 13 · 05/12 周一 — 纹理与贴图 🟡 ​

预估时长： 1.5~2h

学习内容：

  • TextureLoader 加载图片纹理

  • 纹理重复与偏移：wrapS/wrapT、repeat、offset
  • 纹理压缩格式（KTX2/Basis）— 企业项目必知
  • 纹理过滤：minFilter、magFilter
  • 企业应用场景：地面贴图、建筑外立面、设备面板

学习资料：

  • Three.js 手册 - 纹理：https://threejs.org/manual/#en/textures
  • TextureLoader 文档：https://threejs.org/docs/#api/en/loaders/TextureLoader
  • KTX2Loader 文档：https://threejs.org/docs/#examples/en/loaders/KTX2Loader

纹理资源：

  • Three.js内置纹理：three/examples/textures/
  • ambientCG（免费PBR纹理）：https://ambientcg.com/
  • Poly Haven（免费HDRI和纹理）：https://polyhaven.com/textures

实操任务：

  1. 给一个立方体的每个面贴不同纹理
  2. 给地面贴一个重复的网格纹理
  3. 使用法线贴图让平面看起来有凹凸感

⚠️ 常见坑点提醒 ​

坑解决方案
模型加载后看不见检查相机位置、模型缩放比例（GLTF模型可能很大或很小）
阴影不显示renderer.shadowMap.enabled = true + 光源和物体都要开启castShadow/receiveShadow
纹理加载失败检查路径、CORS策略、图片格式
OrbitControls不生效需要在动画循环中调用 controls.update()（如果开了damping）
InstancedMesh颜色不生效需要材质支持 vertexColors，或在着色器中处理
大屏模糊检查 setPixelRatio 和 CSS 缩放
GLTF模型太大使用DRACOLoader压缩，或让设计师导出时优化
